# Mock Data Policy

Mock records are offline software fixtures only.

They are used to test code paths, CI behavior, artifact generation, and schema handling without requiring API keys, live internet access, wet-lab data, or a compiled PhysiCell executable.

Mock records are not:

- real scholarly citations
- manuscript evidence
- wet-lab validation
- PhysiCell simulation evidence
- substitutes for user-supplied validation data

Mock records must be labeled as fixtures in outputs and documentation. They must not be used to claim that the biology is validated.
